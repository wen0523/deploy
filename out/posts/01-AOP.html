Spring Boot AOP 完整指南（一）：基础概念和环境搭建

一、什么是 AOP

AOP（Aspect-Oriented Programming，面向切面编程）是一种编程范式，用于将横切关注点从业务逻辑中分离出来，提高代码的模块化程度。

为什么需要 AOP

在实际开发中，我们经常遇到这样的场景：

  • 需要在多个方法中记录日志
  • 需要在方法执行前进行权限校验
  • 需要统计方法执行时间
  • 需要统一处理异常
  • 需要管理事务

这些功能与核心业务逻辑无关，但又散布在各个业务方法中，导致代码重复、难以维护。AOP 就是为了解决这个问题而诞生的。

核心概念

横切关注点（Cross-cutting Concerns）

分散在应用各处的通用功能，如日志记录、事务管理、权限检查、性能监控等。

切面（Aspect）

模块化的横切关注点，使用 @Aspect 注解标识。

连接点（Join Point）

程序执行过程中的某个特定点，在 Spring AOP 中特指方法的执行。

切入点（Pointcut）

匹配连接点的表达式，定义了在哪些方法上应用切面逻辑。

通知（Advice）

在切入点执行的具体操作，主要有以下类型：

  • @Before：前置通知，目标方法执行前执行
  • @After：后置通知，目标方法执行后执行（无论是否异常）
  • @AfterReturning：返回通知，目标方法正常返回后执行
  • @AfterThrowing：异常通知，目标方法抛出异常后执行
  • @Around：环绕通知，包围目标方法执行，最强大的通知类型

目标对象（Target Object）

被切面所通知的对象，即被代理的业务类。

织入（Weaving）

将切面应用到目标对象并创建代理对象的过程。

二、环境搭建

1. 添加依赖

在 Spring Boot 项目中，只需添加一个 starter 依赖：

<dependencies>
    <!-- Spring Boot Web -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- Spring Boot AOP -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-aop</artifactId>
    </dependency>

    <!-- Lombok（可选） -->
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>

spring-boot-starter-aop 会自动引入：

  • spring-aop：Spring AOP 核心包
  • aspectjweaver：AspectJ 织入包
  • spring-aspects：Spring 对 AspectJ 的集成支持

2. 启用 AOP

Spring Boot 会自动启用 AOP，无需额外配置。如果需要自定义配置：

@SpringBootApplication
@EnableAspectJAutoProxy(proxyTargetClass = true)  // 可选
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

三、AOP 的工作原理

Spring AOP 基于动态代理实现：

  1. JDK 动态代理：当目标对象实现了接口时，Spring 使用 JDK 动态代理
  2. CGLIB 代理：当目标对象没有实现接口时，Spring 使用 CGLIB 代理

通过 @EnableAspectJAutoProxy(proxyTargetClass = true) 可以强制使用 CGLIB 代理。

四、小结

本文介绍了 AOP 的基础概念和环境搭建：

  • AOP 的核心价值：将横切关注点与业务逻辑分离，提高代码的模块化和可维护性
  • 核心概念：切面、连接点、切入点、通知、目标对象、织入
  • 环境搭建：只需添加 spring-boot-starter-aop 依赖即可
